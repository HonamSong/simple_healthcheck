# health-server 프로젝트 작업 규칙

## 세션 시작 시 (필수)

이 디렉토리에서 작업을 시작하면 **가장 먼저 history.md를 읽어** 다음을 파악한다:
- 프로젝트의 현재 상태 (마지막 작업 내용)
- 배포 환경/이미지 URI
- 미해결 후속 작업
- 이전 설계 결정의 근거

그 후 사용자의 지시를 받아 작업을 시작한다.

## 작업 종료 시 (필수)

의미 있는 변경(코드 수정, 빌드, 배포, 설정 변경 등)이 끝날 때마다
**history.md 맨 위에 새 엔트리를 추가**한다. 형식:

```
## [YYYY-MM-DD] 작업 제목

- **의도**: 왜 이 작업을 했는지
- **변경**: 어떤 파일을 어떻게 수정했는지
- **결과/검증**: 빌드/테스트/배포 결과
- **후속**: 다음에 이어서 할 일 (있다면)
```

단순 조회(읽기/질문 응답)만 한 경우에는 기록하지 않는다.
동일 세션 내 작은 수정이 여러 번 반복되면 하나의 엔트리로 묶는다.
